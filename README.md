CS 255 Portfolio Reflection - DriverPass Project

Project Summery
The DriverPass project involved designing a system for a client named DriverPass, a company that provides training services to help students pass their driving exams. The client wanted a web-based system that would allow users to register, schedule driving lessons, take online practice tests, and track their progress. The system also needed administrative features for managing users, scheduling, and maintaining up-to-date DMV information.

What I Did Well
One of my strongest areas in this project was clearly defining both functional and nonfunctional requirements in the business requirements document. I ensured that each requirement followed the correct format (e.g., “The system shall…”) and was specific and testable.

I also did well translating those requirements into system design artifacts, such as UML diagrams, making sure they aligned with the client’s needs and were easy to understand for both technical and nontechnical stakeholders.

What I Would Revise
If I could revise one part of my work, I would improve the level of detail in my UML diagrams, especially the sequence and activity diagrams. While they were correct, I would enhance them by adding more detailed interactions and edge cases, such as error handling and alternate flows.

This would make the design more complete and better prepared for real-world implementation.

Interpreting User Needs
To interpret the user’s needs, I carefully analyzed the client interview and identified key system functionalities such as scheduling, user account management, and online testing. I then translated these needs into both functional requirements and system models.

Considering user needs is critical because if the system does not meet user expectations, it may fail regardless of how well it is built technically.

My Approach to Software Design
My approach to designing software begins with understanding the problem and gathering clear requirements from stakeholders. From there, I break the system into smaller components and use modeling techniques such as UML diagrams (use case, activity, and sequence diagrams) to visualize the system.

In the future, I will continue using structured approaches like the Software Development Life Cycle (SDLC), along with iterative design and feedback loops. I would also incorporate techniques such as prototyping and user testing earlier in the process to validate assumptions and improve usability.
